# 이게머고? 프로젝트

## 스택
- Frontend: Next.js (TypeScript, App Router, Tailwind CSS)
- DB/Auth: Supabase (구글/카카오/네이버 소셜로그인)
- 배포: Vercel
- CDN: Cloudflare

## 사이트 컨셉
- 벤치마크: thisiswhyimbroke.com
- 이름: 이게머고?
- 타겟: 20~40대, 밈 친화적, 얼리어답터
- 톤: 힙하고 위트있는 긱 스페이스, 촌철살인 드립
- 비즈니스 모델: 쿠팡파트너스 + 알리익스프레스 어필리에이트

## 디자인 규칙
- 배경: #FFFFFF
- 텍스트: #111111
- CTA/포인트 컬러: #FF5A00 (가격, 버튼에만 제한적 사용)
- 레이아웃: 풀와이드 그리드 (데스크톱 3~4열, 모바일 1열)
- 사이드바 없음

## 카드 구조 (5층)
1층: 카테고리
2층: 1:1 정사각형 이미지 (Cover 정렬)
3층: 드립형 제목 (촌철살인 한 줄)
4층: 가격
5층: 오렌지 외부링크 버튼 [구경하러 가기] (새 창)

## 카테고리
- 순한맛: 이게 머고?
- 보통맛: 이게? 머고???
- 매운맛: 이게??? 머고???????

## 상품 상세 페이지
- 경로: /product/[id]
- 목적: 공유 링크용 페이지 (SNS/카톡 공유 시 이 페이지로 진입)
- 상단: 해당 상품을 크게 표시
  - 큰 이미지 (좌측 또는 상단)
  - 카테고리 뱃지
  - 드립형 제목 (크게)
  - 가격
  - [구경하러 가기] 버튼 (오렌지, 새 창)
  - 공유 버튼 (링크 복사 등)
- 하단: 다른 상품 그리드 + 무한 스크롤 ("이것도 머고?" 섹션)
- OG 태그 포함 (카카오/트위터 공유 미리보기용)

## 핵심 기능 (우선순위)
1. 풀와이드 그리드 + 카드 컴포넌트
2. 무한 스크롤
3. 소셜 로그인 (Supabase Auth)
4. 찜하기 (하트, 마이페이지)
5. 상품 공유 기능
6. 쿠팡/알리 API 연동 (나중 단계)

## 작업 방식 (중요)
- 나는 개발 경험 없는 완전 초보
- 단계별로 확인하면서 진행할 것
- 한 번에 너무 많이 바꾸지 말 것
- 각 단계마다 뭘 했는지 한국어로 설명할 것
- 에러 나면 혼자 못 고치니 우회 방법 먼저 제시할 것
- 모든 대화는 한국어로
